    It appears to me that you imported the groups from this profile individually and in an order that is not consistent with the order that they were in in the profile that you imported. When importing a profile, you should always choose the option to import the whole thing all at once instead of importing the groups one by one, as this can cause the links between icons to become broken when importing icons, groups, or profiles that were exported by someone who is using a pre-v7.0.0 version of TellMeWhen. TellMeWhen v7.0.0, which is currently in alpha, has mechanisms to prevent these links from being broken when importing icons and groups individually, but it requires that the source string and the client that is importing the string both be running this version, and this is not the case here.

    Sorry about that - my original comment was not correct about the cause of the issue. What is happening here is simply that you have meta icons that are trying to check other icons that don't exist (in certain specs), which is exactly what these warnings are there to warn you about. They aren't errors - just warnings that something might be happening that you didn't intend. In this case, there is nothing wrong with this setup, so you can safely disable the warnings.

    You can disable these warnings by disabling the "Warn about invalid icons" option in TMW's main options.
