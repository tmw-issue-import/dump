Icons with Absent condition set are dimming after reloadui or zoning #574

  • Defect
  • Fixed
Closed
Assigned to
Cybeloras
  • ywfn created this issue Sep 2, 2012

    To reproduce:

    Create a group containing two icons, both set for buff on player, I use the spell Arcane Brilliance.
    Set one icon to 100% opacity for Present and 100% opacity for Absent.
    Set the other icon to 100% opacity for Precent, and Absent unchecked.
    Exit TMW config mode.
    Cast Arcane Brilliance.
    Both icons will be at full "brightness" (I'm not sure opacity is involved, although it might be, it's hard to tell for me).
    Next, reloadui or change zones.  One icon (the one with an Absent condition set) will switch to a lower "brightness" level.  The only way to fix it is to relog, or enter and exit TMW config mode.

    Here's my export for the test group I was using for the above two icons:

    Did some more testing, the problem only happens if the relevant buff (Arcane Brilliance in the example I gave) is up when you reloadui or zone.
