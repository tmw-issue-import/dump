  • Cybeloras posted a comment Aug 22, 2015

    Your code is severely flawed.

    • sforwrath is defined as a function that takes no parameters, yet you pass it one in both places where you call it.
    • The arguments that you do pass in those locations are undefined.
    • There are two variables with the same name as those undefined arguments defined within that function, but they are local to that function and thus cannot be referenced outside of it.
    • You compare the return values of both function calls, but sforwrath doesn't return anything
    • sforwrath creates 9 functions every time it is called. In the case of Lua conditions on an icon, this is as fast as your update interval, which by default is 16 times per second. This is going to create a massive amount of memory churn. Furthermore, those functions are only ever called one time after they are defined, so there is no reason for them to be functions in the first place.
    • The following variables are all leaked globals, which is just bad practice in general, and may cause issues with your code's functionality: sfdmg, spellID, maxRange, minRange, castTime, icon, rank, name, sfcasttime, wcasttime, cpower, edirection, sffem, wfem, getmas, gethas, wdmg

    Here is the function:

    function TMW.CNDT.Env.GetBalanceSpellDPS(spellID, baseDamage)

	local _, _, _, castTime = GetSpellInfo(spellID)
	castTime = castTime / 1000
	
	local cpower = UnitPower("player", 8)
	local edirection = GetEclipseDirection()
	local mastery = GetMasteryEffect() + 30


	local fem
	if edirection == "sun" then
		fem = math.asin(cpower/105) + math.pi/10 * castTime
	else
		fem = math.pi - math.asin(cpower/105) + math.pi/10 * castTime
	end
	fem = math.max(-100, math.min(100, 105*math.sin(fem)))

	return baseDamage*(1+(((fem)+100)/2/100*mastery)/100)/castTime
end

    Here's the code for the Lua condition:

    GetBalanceSpellDPS(5176, 140.4) > GetBalanceSpellDPS(2912, 224.6)
