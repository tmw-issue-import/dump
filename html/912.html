  • Cybeloras posted a comment Jan 13, 2014

    I can't see myself implementing this. The purpose of the currency conditions was to add an easy way to notify you when you have hit, or are arbitrarily close to hitting, the cap on one of your currencies like Honor or Justice. If you would like to see exactly how much you have then you should just open up your character pane and check. If you use a LDB client addon like Titan Panel or FuBar then there are plenty of plugins to show your currencies as well: http://www.curse.com/search/addons?game-slug=wow&search=currency

    There might be a currency icon type at some point in the future, but I wouldn't count on it happening any time soon.

  • Cybeloras posted a comment Jan 14, 2014

    Well, its an issue of feature creep. There are already quite a few things in TMW that, in hindsight, I would have rather not ever implemented. Colors are one of them. DogTag, although it is vastly superior to TMW's old, pathetic dynamic text system, is another - there are a plethora of issues with it that make it a bad fit for TMW, but unfortunately, there is no better system out there at present without writing my own.

    Adding these as an extension of DogTag itself (as opposed to the TMW-specific tags that TMW adds) would be possible, but it is a pretty inactive project at this point and creating an entirely new library just for currency DogTags might be a bit overkill. I can give you a Lua snippet for TMW that will add a currency DogTag when I get home today.

    The ultimate dream would be to have a unified system of data sources that can be implemented as icon types, as conditions, and as text substitutions (like DogTags), but that would be such a huge undertaking and I'm not sure I have enough time to do something like that anymore.

  • Cybeloras posted a comment Jan 14, 2014

    Import this snippet to get a currency DogTag. You must provide the ID as the argument - you can't track by name. Get the ID from the Wowhead URL:

    http://www.wowhead.com/currency=395

    ^1^T^SName^SCurrency~`DogTag ^SCode^Slocal~`DogTag~`=~`LibStub("LibDogTag-3.0",~`true)~J DogTag:AddTag("TMW",~`"Currency",~`{~J ~`~`~`~`~`~`~`~`code~`=~`function(id)~J ~`~`~`~`~`~`~`~`~`~`~`~`return~`select(2,~`GetCurrencyInfo(id))~`or~`0~J ~`~`~`~`~`~`~`~`end~J ~`~`~`~`~`~`~`~`,~J ~`~`~`~`~`~`~`~`arg~`=~`{~J ~`~`~`~`~`~`~`~`~`~`~`~`'id',~`'number',~`'@req',~J ~`~`~`~`~`~`~`~`},~J ~`~`~`~`~`~`~`~`ret~`=~`"number",~J ~`~`~`~`~`~`~`~`events~`=~`"CURRENCY_DISPLAY_UPDATE",~J ~`~`~`~`~`~`~`~`doc~`=~`"Returns~`the~`amount~`of~`the~`specified~`currency~`that~`you~`have.~`Input~`must~`be~`an~`ID~`-~`you~`can~`get~`the~`ID~`from~`the~`Wowhead~`URL~`for~`the~`currency's~`Wowhead~`page.",~J ~`~`~`~`~`~`~`~`example~`=~`'[Currency(395)]~`=>~`"4000"',~J ~`~`~`~`~`~`~`~`category~`=~`"CUSTOM",~J }) ^t^N70047^S~`~| ^Scodesnippet^^

    Thanks so much :D .. It's great, and I get the dogtag stuff much better now also ..

    I did a quick path retrieval for the basic pve and pvp icon textures, and thought I'd share it here for anyone else popping in for the same info :-)

    PVE:
    - Interface\Icons\pvecurrency-justice (ID for DogTag is 395)
    - Interface\Icons\pvecurrency-valor (ID for DogTag is 396)

    PVP: (change Alliance to Horde if you need the Horde version - currency id is the same for both factions)
    - Interface\Icons\PVPCurrency-Honor-Alliance (ID for DogTag is 392)
    - Interface\Icons\PVPCurrency-Conquest-Alliance (ID for DogTag is 390)

    Just paste the required path in the Custom Texture box of the icon in question.
