Add Set Bonus Detection #852

  • Enhancment
  • Declined
Closed
Assigned to
Cybeloras
  • Torchler created this issue Aug 6, 2013

    Ovale for example allows as a condition to check what set bonuses are available based on the gear equiped.

    Their code for this can be found in the 'OvaleEquipement.lua'.

    I know i would find this handy myself as i build a lot of rotation helpers and share them amongst my friends who want them.

    I would be more than happy each patch to provide all the item id's for the gear and it could be limited to gear per expansion of course same as Ovale has done.

  • Torchler     added the tags New Enhancment Aug 6, 2013
  • Cybeloras posted a comment Aug 6, 2013

    I just can't see a feature like this being used enough to justify its existence. There are two options for this:

    1. Let you maintain the list of items in each set and trust that you will keep it updated each and every patch from here on out. Problem: I don't know you at all so I don't know how long you would actually stick around for. Don't take this the wrong way. I would also have to merge in your list every time you update it which can be a bit of a hassle especially since TMW is extremely close to reaching the state where it only needs to be updated once per expansion.
    2. Use LibPeriodicTable-3.1 to get the data for sets. This requires less maintenance, but it also requires a larger justification for its inclusion into the AddOn because of the not-insignificant amount of memory usage increase it would cause. It also relies on that library being updated and maintained in a timely fashion in the future, and while it is much more likely than a custom solution, it is still an unknown factor.

    This feature would essentially be used to track something that would only ever change one time in a character's life - you only acquire each tier set once, and after you get it, you are never going to wear a lesser tier set. It makes a lot more sense to me to just give you friends several icons for each tier bonus and have them enable/disable the ones that do/don't apply to their current gear configuration.

    Conditions should be used for things that will change frequently (or at the very least, will change more than once), and this one just doesn't fit the bill.
